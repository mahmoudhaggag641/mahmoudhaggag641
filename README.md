### 🤖 &nbsp;AI as part of my workflow

I treat modern AI tools (Claude, Copilot, ChatGPT) as a regular part of how I think and code — pairing with them on analysis, scaffolding, testing, and review. They sharpen my speed without replacing the engineering judgment behind what gets built.

| Where I use AI | What it looks like |
| --- | --- |
| **📐 &nbsp;Thinking &amp; design** | Working through architecture trade-offs, refining specs, sketching ERDs and API contracts. |
| **⚡ &nbsp;Writing code** | Scaffolding boilerplate, refactoring legacy modules, generating types and DTOs at higher pace. |
| **🧪 &nbsp;Testing &amp; review** | Generating PHPUnit / Pest tests, scanning for security smells, N+1s, and edge cases. |
| **🚀 &nbsp;Tooling &amp; ops** | Dockerfiles, CI/CD pipelines, runbooks, keeping API docs in sync with the codebase. |
| **🐛 &nbsp;Debugging** | Pattern-matching across logs and exploring hypotheses faster on tricky bugs. |

> AI suggests — I validate. Architecture, security, and judgment calls stay mine.

### 💡 &nbsp;Engineering principles

- 🧹 &nbsp;**Clean code over clever code** — readability beats brevity; the next engineer matters
- 🧪 &nbsp;**Tests as documentation** — TDD where the domain is tricky, pragmatic coverage elsewhere
- 🤝 &nbsp;**APIs are contracts** — versioned, documented, and backward-compatible by default
- 🔐 &nbsp;**Security from day one** — input validation, least privilege, parameterized queries
- 🚀 &nbsp;**Small PRs, fast feedback** — predictable shipping beats heroic refactors
- 🎓 &nbsp;**Code review is teaching** — in both directions
